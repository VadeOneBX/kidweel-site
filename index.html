Positioning

Automation is moving from recommendation to action. That shift changes what failure costs. A bad suggestion wastes a review cycle. A bad action executes.

Finance is the pressure chamber for this problem — small errors compound, and there is no clean undo once an order fills. Kidweel was built there first because the cost of getting it wrong is immediate and measurable. The system underneath is portable: any workflow where a tool can act on a person's behalf needs the same thing — context verified, policy checked, downside bounded, before anything moves.

The core artifact is a bounded decision system: inputs, deterministic gates, human review, evidence trails, and explicit authority boundaries.

Proof

Deterministic reject, on the record

A capstone run can close on a clean rejection: risk/reward and expected-value gates fail on the candidate, no broker submission occurs, and the audit trail writes the outcome to logs. The gate doing its job is not a bug. Rejection is a first-class output.

Advisory layer, not a black box

The capstone control layer adds structured advisory decisions — Confirm, Reject, Watch, Insufficient Data — on top of already-running infrastructure, with a hard transport gate that blocks execution before any assessment runs. Tools suggest. Rules approve.

Authority sits outside the model

Tool descriptions and implementation behavior can diverge in real systems. Kidweel treats that as a design problem: a tool's description is never enough. Authority, review, and evidence have to sit outside the model-facing description.
